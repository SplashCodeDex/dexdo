# Coding & Quality Standards

## General Principles
-   **Measure Twice, Code Once**: Planning precedes implementation.
-   **Clean Code**: Descriptive naming, modular architecture, and thorough documentation.
-   **Performance**: Optimize animations and data fetching for a smooth experience.

## Flutter Standards
-   **Linting**: Adhere to `flutter_lints` recommendations.
-   **State Management**: Use `Provider` consistently; avoid unnecessary rebuilds.
-   **Assets**: Use SVGs for icons and resolution-aware variants for images.

## Web Standards
-   **React**: Use functional components and hooks.
-   **Styling**: Use TailwindCSS 4 utility classes; avoid inline styles.
-   **Types**: TypeScript is mandatory for all new web components.

## Quality Gates
-   **Testing**: Unit and widget tests for critical business logic.
-   **Review**: Static analysis and manual review of UI/UX for "premium" feel.
